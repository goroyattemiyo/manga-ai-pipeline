このプロンプトをそのままAIに貼ってください。
ChatGPT をメインに設計しています。Claude・Gemini向けの注記は末尾にあります。

### ステップ1.5：ページ設計（密度チェック）

**ここをスキップするとコマが薄くなります。必ず実施してください。**

#### ピーク分析

話全体の「笑いのピーク」「感情のピーク」を特定してください。

```
笑いのピーク：（一番おかしい場面・1〜2行で）
感情のピーク：（一番気持ちが動く場面・1〜2行で）
ピークページ：P○〜P○
```

#### ページリズム表

全ページのコマ数・役割・密度を先に決めてください。

```
| ページ | コマ数 | 役割             | 密度   |
|--------|--------|------------------|--------|
| P1     |        | 導入・状況提示   | 低〜中 |
| P2     |        | 盛り上がり前半   | 中     |
| P3     |        | ピーク前半       | 高     |
| P4     |        | ピーク後半       | 高     |
| P5     |        | 落差・切り替わり | 中     |
| P6     |        | オチ             | 中〜高 |
```

※ページ数が異なる場合は行を増減してください。

#### コマ数のルール

- 導入ページ（P1）：4コマ以上を基本とする。3コマは原則禁止
- ピークページ：4〜5コマ。密度を上げて感情・笑いを積み上げる
- オチページ（最終ページ）：大ゴマ1＋小コマ2〜3。大ゴマでオチを決める
- 3コマ以下にする場合：必ず理由を明記する（大ゴマで見せる・間を取るなど）
- 全ページの合計コマ数：【ページ数×3.5〜4.5を目安。例：6ページなら22〜28コマ】

#### ページ設計のNG

- P1・P2が両方3コマ以下（導入が薄すぎる）
- ピークページが3コマ以下（一番おいしい場面を使い切れていない）
- 全ページ同じコマ数（リズムが単調になる）
- 情報量の少ないコマを並べてページを埋めるだけになる

#### 設計確認

ページリズム表を出したあと、以下を自問してください。

```
□ ピークに十分なページ・コマが割かれているか
□ 1コマに「起きていること」が明確にあるか
□ コマとコマの間に「読者が反応する間」があるか
□ サブキャラのリアクションコマが各ページに1つ以上あるか
□ P1だけで話の状況・キャラ・雰囲気が伝わるか
```

確認が取れたらステップ2へ進んでください。

### ステップ2：ネーム（コマ台本）

ステップ1.5のページリズム表に従ってネームを書いてください。
リズム表から外れるコマ数にする場合は理由を担当者メモに明記してください。

```
## ○話「タイトル」ネーム

### P1（1ページ目）— [役割] [コマ数]コマ

コマ1　[size: large / medium / small]
[状況・アングル・背景を1行で]
- 【キャラA】：「セリフ」
- 【キャラB】：（ジェスチャー描写）
- 効果：[漫画効果があれば]

コマ2　[size: large / medium / small]
...

### P2（2ページ目）— [役割] [コマ数]コマ
...

### 最終ページ — [オチ] [コマ数]コマ

ラストコマ　[size: large]
[状況]
- 【キャラA】：「セリフ」
- 【キャラB】：（ジェスチャー描写）
【キャラCがいる場合：】
- 【キャラC】（遠くで）：「……セリフ。」

### 担当者メモ
- コマ数合計：（○コマ）
- リズム表からの変更点と理由：
- 再利用注意ネタ：
- 次話への引き：
- 設定資料への追記が必要な新設定：
```

#### コマ密度チェック

各コマを書いたあと、以下を確認してください。

```
□ そのコマで「何が起きているか」が1行で言えるか
□ セリフとジェスチャーが同じことを言っていないか（冗長）
□ 前のコマと状況がほぼ同じになっていないか（コマを無駄にしている）
□ サブキャラが2コマ以上連続で無反応になっていないか
□ 主人公のセリフが1コマに2行以上になっていないか（長すぎ）
```

確認が取れたらステップ2.5へ進んでください。

### ステップ2.5：画像生成用YAML出力

ネームが確定したら、画像生成AI向けのYAMLを出力してください。

```yaml
episode: [話数]
title: [タイトル]
total_pages: [ページ数]

references:
  characters:
    - id: [chara_a]
      image: [chara_a_ref.png]
      note: "[キャラAの外見・頭身・衣装の基準]"
    - id: [chara_b]
      image: [chara_b_ref.png]
      note: "[キャラBの外見の基準]"
  props:
    # 話に固有のアイテムがある場合のみ追加
    # 不要な場合は props: []
  backgrounds:
    # 固有の背景参照が必要な場合のみ追加
    # 不要な場合は backgrounds: []

global_ng:
  - yaml未記載のキャラ・アイテム・背景要素を補完しない
  - 【キャラAのNG1】
  - 【キャラAのNG2】
  - 【キャラBのNG1】
  - 【その他の作品全体のNG】

pages:
  - page: 1
    header: "[作品タイトル] EP.[話数]　[話タイトル]"
    layout: [コマ数と配置の説明]
    frame: black
    scene: [場所・時間帯]
    characters_in_page: [登場キャラのidリスト]
    references_required: [このページで必要な参照画像idリスト]
    page_ng: [このページ固有のNG。なければ []]
    panels:
      - panel: 1
        size: [large / medium / small]
        frame: [black / white / black_flash]
        characters: [このコマの登場キャラ]
        action: [キャラの動作・状況を1行で]
        expression: [表情・感情]
        dialogue: "セリフ"
        narration: "ナレーション"
        effects: []
        bg: [背景の説明]
```

#### frameキーのルール

- 現在シーン・日常シーン：frame: black
- 回想中のコマ：frame: white
- 回想終了の暗転コマ：frame: black_flash

#### YAML生成時のルール

- references.charactersには登場キャラを全員入れる
- global_ngには作品全体のNGを必ず入れる
- page_ngにはそのページで補完されやすいリスクを書く
- P1にのみheaderフィールドを入れる（P2以降は不要）

### NGチェック（生成後に必ず確認）

#### ページ設計チェック

```
□ ステップ1.5のページリズム表を出したか
□ P1が3コマ以下になっていないか
□ ピークページが3コマ以下になっていないか
□ 合計コマ数が目安の範囲内か
□ 全ページ同じコマ数になっていないか
□ リズム表から外れた場合、理由を担当者メモに書いたか
```

#### ネーム内容チェック

```
□ サブキャラのセリフが多すぎないか
□ 設定説明だけでページが埋まっていないか
□ オチが「夢でした」になっていないか
□ 主人公がネガティブで沈み続けていないか
□ 同じネタを前の話と重複して使っていないか
```

#### YAML出力チェック

```
□ references.charactersに全キャラが入っているか
□ global_ngに作品全体のNGが入っているか
□ P1にheaderフィールドが入っているか
□ 回想コマにframe: whiteが入っているか
□ 回想終了コマにframe: black_flashが入っているか
```

## AI別の注意点

### ChatGPT（GPT-5.5 / GPT Image 2）推奨
- このプロンプトをそのまま貼って会話を始めてください
- GPTsのSystem Promptに入れると毎回貼らずに済みます
- YAMLを画像生成に渡す手順は `05_image_generation/gpts_system_prompt.md` を参照してください

### Gemini
- このファイルのGitHub URLをGeminiに渡してください
  https://raw.githubusercontent.com/goroyattemiyo/manga-ai-pipeline/main/03_scenario/scenario_prompt.md
- 「このファイルの手順に従って【話数】話のネームを作ってください」と伝えてください
- キャラの一貫性が弱い場合は、CHARACTER_BIBLEのURLも一緒に渡してください

### Claude
- このファイルをプロジェクトに登録してください
- CHARACTER_BIBLE・WORLD_BIBLE・SERIES_GUIDEも同じプロジェクトに登録すると設定を毎回読み込みます
- 画像生成はできないため、YAMLをChatGPT / GeminiのGPTsに渡してください
